# Deployment Scripts Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship a minimal VPS deployment workflow for Atrium: an on-server `bin/deploy` shell script that builds an Elixir release under `asdf`, a `bin/deploy_remote` SSH wrapper, an `Atrium.Release` module for running migrations (global + tenant), an nginx config template, and release config files generated by `mix phx.gen.release`.

**Architecture:** Modeled on the alldoqexchange deployment but stripped of app-specific complexity (no DoqBuilder frontend clone/embed, no PDF.js setup, no per-environment stop scripts, no tula port special case). Shared-directory + symlink rotation with `keep_releases` cleanup; daemon restart via `bin/atrium daemon`. Nginx reverse proxies to the Bandit-served Phoenix app.

**Tech Stack:** Bash, asdf, Elixir releases (`mix release`), Triplex, nginx.

**Spec:** `docs/superpowers/specs/2026-04-20-deployment-and-encrypted-file-documents-design.md` — Part 2.

```elixir
defmodule Atrium.Release do
  @moduledoc "Release-time tasks: migrate the global schema, then all tenant schemas."
  @app :atrium

  def migrate do
    load_app()

    for repo <- repos() do
      {:ok, _, _} = Ecto.Migrator.with_repo(repo, &Ecto.Migrator.run(&1, :up, all: true))
    end

    migrate_tenants()
  end

  end

    Application.ensure_all_started(:triplex)

    tenants =
      try do
        Atrium.Tenants.list_tenants()

        UndefinedFunctionError -> []
      end

    Enum.each(tenants, fn tenant ->
      slug = tenant.slug

      _ = Triplex.migrate(slug)
    end)
  end

  defp repos, do: Application.fetch_env!(@app, :ecto_repos)
  defp load_app, do: Application.load(@app)
end
```

## Task 3: bin/deploy (on-server script)

**Files:**
- Create: `bin/deploy`

- [ ] **Step 1: Create the script**

Create `bin/deploy` (make executable):

```bash
#!/bin/bash
set -e

# ==============================================================================
# Atrium deployment script (runs on the target VPS).
# Clones the repo, builds a prod release under asdf, rotates symlinks, runs
# migrations (global + per-tenant), restarts the daemon, cleans up old releases.
# ==============================================================================

DEPLOY_USER="${DEPLOY_USER:-$(whoami)}"
DEPLOY_PATH="${DEPLOY_PATH:-/var/www/atrium}"
REPO_URL="${REPO_URL:-git@github.com:yourorg/atrium.git}"
BRANCH="${BRANCH:-main}"
KEEP_RELEASES="${KEEP_RELEASES:-5}"
APP_NAME="${APP_NAME:-atrium}"
MIX_ENV="${MIX_ENV:-prod}"
PORT="${PORT:-4000}"

SHARED_DIRS="uploads logs"
SHARED_FILES=".env"

RED='\033[0;31m'
GREEN='\033[0;32m'
YELLOW='\033[1;33m'
NC='\033[0m'

log_info()  { echo -e "${GREEN}[INFO]${NC}  $1" >&2; }
log_warn()  { echo -e "${YELLOW}[WARN]${NC}  $1" >&2; }
log_error() { echo -e "${RED}[ERROR]${NC} $1" >&2; }

command_exists() { command -v "$1" >/dev/null 2>&1; }

# --- asdf -------------------------------------------------------------------

install_asdf() {
  if [ -d "$HOME/.asdf" ]; then
    log_info "asdf already installed"
    return 0
  fi
  log_info "Installing asdf..."
  git clone https://github.com/asdf-vm/asdf.git ~/.asdf --branch v0.14.0
  if [ -f "$HOME/.bashrc" ]; then
    echo '. "$HOME/.asdf/asdf.sh"' >> ~/.bashrc
  fi
  if [ -f "$HOME/.zshrc" ]; then
    echo '. "$HOME/.asdf/asdf.sh"' >> ~/.zshrc
  fi
  . "$HOME/.asdf/asdf.sh"
}

setup_asdf() {
  if [ -d "$HOME/.asdf/shims" ]; then
    export PATH="$HOME/.asdf/shims:$PATH"
  fi
  if [ -f "$HOME/.asdf/asdf.sh" ]; then
    . "$HOME/.asdf/asdf.sh"
  fi
  if ! command_exists asdf; then
    install_asdf
  fi
}

install_asdf_plugin() {
  local plugin=$1
  if asdf plugin list 2>/dev/null | grep -q "^${plugin}$"; then
    log_info "asdf plugin ${plugin} already installed"
  else
    log_info "Installing asdf plugin: ${plugin}"
    asdf plugin add "${plugin}" >&2 2>&1 || true
  fi
}

install_language_versions() {
  local file=$1
  if [ ! -f "$file" ]; then
    log_warn ".tool-versions not found at ${file}"
    return 0
  fi
  while IFS= read -r line; do
    [[ -z "$line" || "$line" =~ ^# ]] && continue
    local plugin version
    plugin=$(echo "$line" | awk '{print $1}')
    version=$(echo "$line" | awk '{print $2}')
    install_asdf_plugin "$plugin"
    if asdf list "$plugin" 2>/dev/null | grep -q "$version"; then
      log_info "${plugin} ${version} already installed"
    else
      log_info "Installing ${plugin} ${version}..."
      asdf install "$plugin" "$version" >&2 2>&1
    fi
  done < "$file"
}

# --- deploy -----------------------------------------------------------------

init_directory_structure() {
  log_info "Initializing directory structure at ${DEPLOY_PATH}"
  mkdir -p "${DEPLOY_PATH}/releases"
  mkdir -p "${DEPLOY_PATH}/shared"
  for dir in $SHARED_DIRS; do
    mkdir -p "${DEPLOY_PATH}/shared/${dir}"
  done
}

create_release() {
  local ts release_path
  ts=$(date +%Y%m%d%H%M%S)
  release_path="${DEPLOY_PATH}/releases/${ts}"

  log_info "Creating new release: ${ts}"
  log_info "Cloning ${REPO_URL} @ ${BRANCH}"
  git clone --depth 1 --branch "${BRANCH}" "${REPO_URL}" "${release_path}" >&2 2>&1

  install_language_versions "${release_path}/.tool-versions"

  # Symlink shared dirs into release
  for dir in $SHARED_DIRS; do
    if [ -d "${release_path}/${dir}" ]; then rm -rf "${release_path}/${dir}"; fi
    ln -sf "${DEPLOY_PATH}/shared/${dir}" "${release_path}/${dir}"
  done

  for file in $SHARED_FILES; do
    if [ -f "${DEPLOY_PATH}/shared/${file}" ]; then
      ln -sf "${DEPLOY_PATH}/shared/${file}" "${release_path}/${file}"
    else
      log_warn "shared/${file} not found"
    fi
  done

  cd "${release_path}"

  log_info "Installing mix/rebar"
  mix local.hex --force >&2 2>&1
  mix local.rebar --force >&2 2>&1

  log_info "Fetching prod deps"
  mix deps.get --only prod >&2 2>&1

  if [ -f "assets/package.json" ]; then
    log_info "Installing npm deps"
    (cd assets && npm ci --production=false >&2 2>&1)
  fi

  log_info "Compiling assets"
  MIX_ENV=prod mix assets.deploy >&2 2>&1

  log_info "Building release"
  MIX_ENV=prod mix release --overwrite >&2 2>&1

  log_info "Running migrations (global + tenants)"
  if ! _build/prod/rel/atrium/bin/atrium eval "Atrium.Release.migrate" >&2 2>&1; then
    log_warn "Migrations reported an error (continuing)"
  fi

  echo "$ts" > "${release_path}/REVISION"
  echo "$ts"
}

update_current_symlink() {
  local release=$1
  local current_path="${DEPLOY_PATH}/current"
  if [ -L "$current_path" ]; then
    log_info "Previous release: $(basename "$(readlink "$current_path")")"
  fi
  ln -sfn "${DEPLOY_PATH}/releases/${release}" "$current_path"
}

cleanup_old_releases() {
  local keep=$1
  local dir="${DEPLOY_PATH}/releases"
  cd "$dir"
  local count
  count=$(ls -1 | wc -l)
  if [ "$count" -gt "$keep" ]; then
    local remove=$((count - keep))
    log_info "Removing ${remove} old release(s)"
    ls -1t | tail -n "$remove" | while read -r old; do
      log_info "  rm -rf ${old}"
      rm -rf "${dir}/${old}"
    done
  fi
}

restart_application() {
  local current_path="${DEPLOY_PATH}/current"
  local bin_path="${current_path}/_build/prod/rel/atrium/bin/atrium"
  local env_file="${DEPLOY_PATH}/shared/.env"

  cd "$current_path"

  if [ ! -f "$bin_path" ]; then
    log_error "Release binary not found at ${bin_path}"
    exit 1
  fi

  if [ -f "$env_file" ]; then
    set -a; source "$env_file"; set +a
  else
    log_warn ".env not found at ${env_file}"
  fi

  export PORT="${PORT:-4000}"
  export MIX_ENV="${MIX_ENV:-prod}"
  export PHX_SERVER="true"

  log_info "Stopping any existing daemon"
  "$bin_path" stop >&2 2>&1 || true
  sleep 2

  local port_pid
  port_pid=$(lsof -ti :${PORT} 2>/dev/null || true)
  if [ -n "$port_pid" ]; then
    log_warn "Port ${PORT} still in use; killing pid ${port_pid}"
    kill "$port_pid" 2>/dev/null || true
    sleep 2
  fi

  log_info "Starting daemon on port ${PORT}"
  "$bin_path" daemon >&2 2>&1
  sleep 3

  if "$bin_path" pid >/dev/null 2>&1; then
    log_info "Running (pid $("$bin_path" pid 2>&1))"
  else
    log_error "Failed to start"
    tail -50 "${current_path}/logs/phoenix.log" >&2 2>&1 || true
    exit 1
  fi
}

rollback() {
  local current_path="${DEPLOY_PATH}/current"
  if [ ! -L "$current_path" ]; then
    log_error "No current release"
    exit 1
  fi

  local current previous=""
  current=$(basename "$(readlink "$current_path")")
  local found=false

  for r in $(ls -1t "${DEPLOY_PATH}/releases"); do
    if [ "$found" = true ]; then previous="$r"; break; fi
    if [ "$r" = "$current" ]; then found=true; fi
  done

  if [ -z "$previous" ]; then
    log_error "No previous release to roll back to"
    exit 1
  fi

  log_info "Rolling back from ${current} to ${previous}"
  update_current_symlink "$previous"
  restart_application
}

deploy() {
  log_info "Deploying ${REPO_URL}#${BRANCH} to ${DEPLOY_PATH}"
  setup_asdf
  init_directory_structure
  local release
  release=$(create_release)
  update_current_symlink "$release"
  cleanup_old_releases "$KEEP_RELEASES"
  restart_application
  log_info "Deployed release ${release}"
}

show_help() {
  cat <<EOF
Atrium deployment

Usage: $0 [deploy|rollback|setup|cleanup|help]

Env vars:
  DEPLOY_PATH    default /var/www/atrium
  REPO_URL       git URL to clone
  BRANCH         default main
  KEEP_RELEASES  default 5
  PORT           default 4000
EOF
}

main() {
  local cmd=${1:-deploy}
  case "$cmd" in
    deploy)    deploy ;;
    rollback)  rollback ;;
    setup)     setup_asdf; [ -f .tool-versions ] && install_language_versions .tool-versions ;;
    cleanup)   cleanup_old_releases "$KEEP_RELEASES" ;;
    help|-h|--help) show_help ;;
    *) log_error "unknown command: $cmd"; show_help; exit 1 ;;
  esac
}

main "$@"
```

```bash
#!/bin/bash
set -e

REMOTE_HOST="${REMOTE_HOST:?REMOTE_HOST is required}"
REMOTE_USER="${REMOTE_USER:-deploy}"
REMOTE_PORT="${REMOTE_PORT:-22}"

REPO_URL="${REPO_URL:-git@github.com:yourorg/atrium.git}"

DEPLOY_PATH="${DEPLOY_PATH:-/var/www/atrium}"
MIX_ENV="${MIX_ENV:-prod}"

GREEN='\033[0;32m'; YELLOW='\033[1;33m'; NC='\033[0m'
log_info() { echo -e "${GREEN}[INFO]${NC} $1"; }
log_warn() { echo -e "${YELLOW}[WARN]${NC} $1"; }

SSH="ssh -p ${REMOTE_PORT} ${REMOTE_USER}@${REMOTE_HOST}"
SCP="scp -P ${REMOTE_PORT}"

ACTION="${1:-deploy}"

log_info "Target: ${REMOTE_USER}@${REMOTE_HOST}:${REMOTE_PORT}  action: ${ACTION}"

log_info "Uploading bin/deploy to remote /tmp/deploy"
${SCP} "$(dirname "$0")/deploy" "${REMOTE_USER}@${REMOTE_HOST}:/tmp/deploy"

log_info "Executing ${ACTION} on remote"
${SSH} "
  chmod +x /tmp/deploy
  REPO_URL='${REPO_URL}' \
  BRANCH='${BRANCH}' \
  DEPLOY_PATH='${DEPLOY_PATH}' \
  MIX_ENV='${MIX_ENV}' \
  KEEP_RELEASES='${KEEP_RELEASES}' \
  PORT='${APP_PORT}' \
  /tmp/deploy ${ACTION}
"

log_info "Done."
```

```nginx
# Atrium nginx site config.
# Place in /etc/nginx/sites-available/atrium.conf, then
# ln -sf /etc/nginx/sites-available/atrium.conf /etc/nginx/sites-enabled/
# nginx -t && systemctl reload nginx

upstream atrium_phoenix {
    server 127.0.0.1:4000;
}

server {
    listen 80;
    listen [::]:80;
    server_name atrium.example.com;

    access_log /var/log/nginx/atrium_access.log;
    error_log  /var/log/nginx/atrium_error.log;

    # Allow 200 MB request bodies (double the 100 MB file-doc cap for headroom)
    client_max_body_size 200M;
    client_body_buffer_size 256k;
    client_body_timeout 600s;
    send_timeout 600s;

    # Health endpoint (no proxy log noise)
    location = /health {
        proxy_pass http://atrium_phoenix;
        access_log off;
    }

    # Static images directly served from shared/uploads (read-only)
    location ~ ^/uploads/documents/[^/]+/images/ {
        root /var/www/atrium/shared;
        try_files $uri =404;
        expires 7d;
        add_header Cache-Control "public";
        access_log off;
    }

    # Main reverse proxy (WebSocket + LiveView upgrade)
    location / {
        proxy_pass http://atrium_phoenix;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header X-Forwarded-Host $host;
        proxy_set_header X-Forwarded-Port $server_port;

        proxy_connect_timeout 300s;
        proxy_send_timeout 600s;
        proxy_read_timeout 600s;

        proxy_request_buffering off;
        proxy_buffering off;
    }
}

# HTTPS block (uncomment once certificates are installed)
# server {
#     listen 443 ssl http2;
#     listen [::]:443 ssl http2;
#     server_name atrium.example.com;
#
#     ssl_certificate     /etc/letsencrypt/live/atrium.example.com/fullchain.pem;
#     ssl_certificate_key /etc/letsencrypt/live/atrium.example.com/privkey.pem;
#     ssl_protocols       TLSv1.2 TLSv1.3;
#
#     client_max_body_size 200M;
#     # ... rest identical to the :80 block above ...
# }
```

**Security note:** the `location ~ ^/uploads/documents/[^/]+/images/` block bypasses Phoenix and serves images directly. Encrypted `/uploads/documents/<tenant>/files/...` paths intentionally fall through to the proxy (where the Phoenix `StaticUploads` plug 404s them). This means even if nginx is misconfigured, the Phoenix guard is still in place.

```markdown
## Deploying

### Prerequisites on the target VPS
- Ubuntu 22.04+ (or similar), `git`, `build-essential`, `libssl-dev`, `autoconf`, `libncurses-dev`, `unzip`
- A `deploy` user with sudo for nginx reload (or run the nginx bits as root)
- Postgres reachable from the app, with a DB and user for Atrium
- `/var/www/atrium` owned by the deploy user (or whatever `DEPLOY_PATH` you pick)

### One-time setup on the server
```
bin/deploy setup          # run ON the server — installs asdf + languages
```

### Shared `.env` (on the server, at `${DEPLOY_PATH}/shared/.env`)
```
DATABASE_URL=ecto://atrium:password@localhost/atrium_prod
SECRET_KEY_BASE=<mix phx.gen.secret>
PHX_HOST=atrium.example.com
PORT=4000
ATRIUM_CLOAK_KEY=<base64-encoded 32 random bytes>
ATRIUM_FILE_ENCRYPTION_KEY=<base64-encoded 32 random bytes>
ATRIUM_UPLOADS_ROOT=/var/www/atrium/shared/uploads
```

Generate each 32-byte key with:
```
openssl rand -base64 32
```

### Deploying from your machine
```
REMOTE_HOST=atrium.example.com \
REMOTE_USER=deploy \
REPO_URL=git@github.com:yourorg/atrium.git \
BRANCH=main \
bin/deploy_remote deploy
```

Or copy `bin/deploy_remote_example` → `bin/deploy_remote_production` and edit, then `bin/deploy_remote_production deploy`.

### Rolling back
```

```

### Nginx
Copy `nginx/atrium.conf` to `/etc/nginx/sites-available/`, edit `server_name` and `root` paths, symlink into `sites-enabled/`, then `sudo nginx -t && sudo systemctl reload nginx`.
```

## Task 7: Verify end-to-end on a scratch VM (manual)

This task is optional but strongly recommended before the first real deploy. No commits.

- [ ] **Step 1: Provision a test VM**

Any small Ubuntu 22.04 VM. Install basics:

```bash
sudo apt update && sudo apt install -y git build-essential libssl-dev autoconf libncurses-dev unzip postgresql nginx
sudo -u postgres createuser -P atrium
sudo -u postgres createdb -O atrium atrium_prod
```

Create a `deploy` user, grant ownership of `/var/www/atrium`.

- [ ] **Step 2: Populate shared/.env on the VM**

SSH as deploy. Create `/var/www/atrium/shared/.env` with the variables from the README.

- [ ] **Step 3: Run `bin/deploy_remote deploy` from your laptop**

From the Atrium repo locally:

```bash
REMOTE_HOST=<vm-ip> REMOTE_USER=deploy REPO_URL=<your repo> bin/deploy_remote deploy
```

Expected:
- SSH succeeds
- asdf installs erlang/elixir/node on first run (long — 5–10 min)
- Release builds
- Migrations run (both global and tenant, though tenant list is empty on a fresh DB — that's fine)
- Daemon starts
- `curl http://<vm-ip>:4000/health` returns 200

- [ ] **Step 4: Smoke-test rollback**

```bash
REMOTE_HOST=<vm-ip> bin/deploy_remote rollback
```

If only one release exists, this errors — deploy once more first, then roll back. Verify symlink flips.

- [ ] **Step 5: Wire up nginx**

On the VM:
```bash
sudo cp /var/www/atrium/current/nginx/atrium.conf /etc/nginx/sites-available/atrium.conf
sudo ln -sf /etc/nginx/sites-available/atrium.conf /etc/nginx/sites-enabled/atrium.conf
sudo nginx -t && sudo systemctl reload nginx
```

Visit `http://<vm-ip>` — should load the app.
